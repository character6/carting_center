## 1. Функциональные требования (ФТ)

### FR-01. Просмотр расписания и слотов
*   **FR-01.1:** Приложение должно отображать список доступных заездов (слотов) на ближайшие 7 дней по умолчанию.
*   **FR-01.2:** Для каждого слота должны отображаться следующие данные, получаемые из API:
    *   Время старта и длительность.
    *   Тип трассы (короткая/длинная).
    *   Имя маршала-инструктора.
    *   Количество свободных мест.
    *   Статус наличия экипировки.
    *   Стоимость заезда.
*   **FR-01.3:** Пользователь должен иметь возможность фильтровать слоты по датам за пределами 7-дневного окна.
*   **FR-01.4:** При отсутствии расписания должен отображаться понятный статус (Empty State): «Пока нет доступных заездов».

### FR-02. Процесс записи (Booking)
*   **FR-02.1:** При выборе слота пользователь обязан выбрать тип экипировки через переключатель: «Своя экипировка» или «Нужен прокат».
*   **FR-02.2:** Пользователь должен самостоятельно указать свой уровень подготовки: «Новичок» или «Опытный».
*   **FR-02.3:** Система должна проверять лимит «не более 8 новичков» на стороне бэкенда. Если лимит превышен, запись отклоняется с сообщением об ошибке.
*   **FR-02.4:** Бронирование считается успешным только после получения положительного ответа от API (гарантия отсутствия двойных броней обеспечивается бэкендом).

### FR-03. Управление записями и отмены
*   **FR-03.1:** Пользователь может отменить свою активную запись через приложение не позднее чем за 2 часа до времени старта.
*   **FR-03.2:** Если до старта осталось менее 2 часов, кнопка отмены в интерфейсе должна быть заблокирована или скрыта.
*   **FR-03.3:** В случае отмены заезда центром (погода/поломка), запись получает статус «Отменён центром» с указанием причины. Повторная запись на этот слот невозможна.

### FR-04. Оплата и финансы
*   **FR-04.1:** Приложение отображает итоговую стоимость заезда (базовая цена + тариф за прокат, если выбран) перед подтверждением.
*   **FR-04.2:** Онлайн-оплата в приложении не производится. Оплата осуществляется строго на месте наличными или переводом.

### FR-05. Уведомления и обратная связь
*   **FR-05.1:** Приложение должно отправлять Push-уведомления при следующих событиях:
    *   Подтверждение записи.
    *   Отмена заезда центром (с причиной).
    *   Напоминание за 1 час до старта.
*   **FR-05.2:** После завершения заезда (статус «Завершен») пользователю предлагается оценить маршала по шкале 1–5 звезд и оставить текстовый отзыв.

### FR-06. Профиль пользователя
*   **FR-06.1:** Регистрация и вход осуществляются по номеру телефона (SMS-код).
*   **FR-06.2:** В профиле отображается история заездов и текущий статус лояльности (бейдж «Постоянный клиент» после N заездов).

## 2. Нефункциональные требования (НФТ)

*   **NFT-01. Производительность:** Время загрузки списка слотов не должно превышать 2 секунд при стабильном соединении.
*   **NFT-02. Надежность:** Приложение должно корректно обрабатывать ошибки API (например, «Мест нет», «Лимит новичков превышен») и показывать понятные пользователю сообщения вместо технических логов.
*   **NFT-03. Безопасность:** Передача данных (особенно номера телефона) должна осуществляться по защищенному протоколу HTTPS.
*   **NFT-04. Совместимость:** Приложение должно корректно работать на iOS (версии 15+) и Android (версии 10+).
*   **NFT-05. UX/UI:** Интерфейс должен быть адаптирован под использование «на ходу» и на улице (крупные кнопки, контрастный текст, минимизация мелких элементов).

## 3. Use Cases (Основные сценарии)

### UC-01: Просмотр и бронирование слота
1.  Клиент открывает раздел «Расписание» и видит слоты на 7 дней вперед.
2.  Выбирает подходящий слот и нажимает «Записаться».
3.  Вводит параметры: выбирает «Прокат» экипировки и указывает уровень «Новичок».
4.  Нажимает «Подтвердить».
5.  Система отправляет запрос в API. Бэкенд проверяет наличие карт и лимит новичков.
6.  При успехе создается бронь, клиент видит экран подтверждения и получает Push-уведомление.

### UC-02: Отмена заезда клиентом
1.  Клиент переходит в раздел «Мои заезды».
2.  Выбирает активную бронь, до старта которой осталось более 2 часов.
3.  Нажимает «Отменить запись».
4.  Система запрашивает подтверждение действия.
5.  После подтверждения бронь аннулируется, место освобождается в расписании.
6.  *Альтернатива:* Если до старта менее 2 часов, кнопка отмены недоступна.

### UC-03: Оценка маршала после заезда
1.  Статус заезда в системе меняется на «Завершен».
2.  Клиенту приходит Push-уведомление с предложением оценить инструктора.
3.  Клиент открывает приложение, видит карточку прошедшего заезда.
4.  Выставляет оценку от 1 до 5 звезд и (опционально) пишет текстовый комментарий.
5.  Нажимает «Отправить». Данные сохраняются в профиле маршала.

## 4. User Stories (От лица Клиента)

1.  **US-01 (Просмотр):** Как клиент, я хочу видеть свободные слоты на неделю вперед с указанием цены и типа трассы, чтобы спланировать свое посещение центра.
2.  **US-02 (Экипировка):** Как клиент, я хочу указать при записи, нужна ли мне прокатная экипировка, чтобы сотрудники центра подготовили шлем и костюм к моему приезду.
3.  **US-03 (Уровень):** Как клиент, я хочу самостоятельно указывать свой уровень подготовки («Новичок» или «Опытный»), чтобы меня распределили в подходящую группу.
4.  **US-04 (Отмена):** Как ответственный клиент, я хочу иметь возможность отменить запись за 2 часа до старта, чтобы освободить место для других, если мои планы изменились.
5.  **US-05 (Напоминание):** Как забывчивый клиент, я хочу получать Push-уведомление за час до заезда, чтобы успеть добраться до центра на окраине города.
6.  **US-06 (Оценка):** Как клиент, я хочу поставить оценку и написать отзыв маршалу после заезда, чтобы поблагодарить его за хорошую инструкцию или указать на недочеты.
7.  **US-07 (Статус отмены):** Как клиент, я хочу видеть статус «Отменён центром» с причиной (например, погода) в своих записях, чтобы понимать, почему заезд не состоялся, и не пытаться записаться на него снова.
