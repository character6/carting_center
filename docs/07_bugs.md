## Баг #1: Двойное бронирование при быстром двойном клике
### Симптом
Пользователь нажимает кнопку «Подтвердить» в модальном окне бронирования. Если нажать быстро дважды, создаются две одинаковые брони в localStorage. В разделе «Мои заезды» отображаются дубликаты одной и той же записи.

### Требование
FR-02.4: Бронирование считается успешным только после получения положительного ответа от API.

### Причина
Кнопка «Подтвердить» не блокировалась во время выполнения запроса к API. Пользователь мог кликнуть несколько раз, и каждый клик отправлял новый запрос на создание брони. Состояние loading не использовалось для блокировки кнопки.

### Исправление
Было (BookingModal.tsx):   

```tsx
<button
  onClick={handleSubmit}
  className="px-5 py-2.5 rounded-xl text-sm font-semibold bg-slate-900 text-white hover:bg-slate-700"
>
  Подтвердить
</button>
```
Стало (BookingModal.tsx):

```tsx
<button
  onClick={handleSubmit}
  disabled={loading}
  className={[
    'px-5 py-2.5 rounded-xl text-sm font-semibold transition',
    loading
      ? 'bg-slate-400 text-white cursor-not-allowed'
      : 'bg-slate-900 text-white hover:bg-slate-700',
  ].join(' ')}
>
  {loading ? 'Записываем...' : 'Подтвердить'}
</button>
```

## Баг #2: Отсутствие валидации дат в фильтре

### Симптом
Пользователь выбирает дату «С» = 10 июля, дату «По» = 5 июля (дата «С» больше даты «По»). Нажимает «Применить». Список слотов пустой, но нет сообщения об ошибке. Пользователь не понимает, что произошло, и думает, что приложение сломалось.

### Требование
NFT-02: Приложение должно корректно обрабатывать ошибки и показывать понятные пользователю сообщения вместо технических логов.

### Причина
В компоненте SlotsList.tsx не было валидации дат перед отправкой запроса. Если from > to, API возвращает пустой массив, но UI не показывает ошибку. Пользователь видит пустой список без объяснения причины.

### Исправление

Было (SlotsList.tsx):

```tsx
const handleApplyFilter = () => {
  setFrom((v) => v);
  setTo((v) => v);
};
```

Стало (SlotsList.tsx):
```tsx
const [dateError, setDateError] = useState<string | null>(null);

const handleApplyFilter = () => {
  if (from > to) {
    setDateError('Дата «С» не может быть позже даты «По»');
    return;
  }
  setDateError(null);
  setFrom((v) => v);
  setTo((v) => v);
};

// В JSX добавлено:
{dateError && (
  <div className="bg-red-50 border border-red-200 text-red-700 rounded-xl p-3 mb-4 text-sm">
    <p className="font-semibold">Ошибка фильтра</p>
    <p>{dateError}</p>
  </div>
)}
```
